Структура проекта сайта:
  makefile -  файл конечной сборки сайта скриптом make а также отправки файлов на web сервер - по сути конечная обертка для главного скрипта сборки  - make.php
  make.php -  главный php скрипт настройки и сборки сайта
  pages    -  папка блочных шаблонов web страниц сайта, в каждый блок будет подставляться компонент
  modules  -  папка с исходниками модулей, которые подставляются в блоки шаблонов страниц сайта
  build    -  папка вывода сборки сайта - сюда попадает конечная файловая система сайта после настройки и сборки , она в итоге отправляется на web-сервер
  any.txt  -  вспомогательный файл заметок разработчика

Структура папки modules:
  папка модуля 1
  папка модуля 2
  ............
  ............
  папка модуля

Структура папки конкретного модуля:
  mod.html - файл html шаблона модуля
  mod.css  - файл стилей модуля
  mod.js   - файл исходного кода клиентской стороны модуля
  mod.php  - файл исходного кода серверной стороны модуля
  mod.sql  - файл исходного кода взаимодейтсвия модуля с базой данных

Мысли:

Разметка окна программы. Конкретный пример разметки характеризуется объектом класса Layout
Layout представляет собой список прямоугольников, делящих окно
w - ширина
h - высота
w и h могут быть либо статически вычислены, либо динамически.
Параметр статически вычислим, если он вычисляется исходя из параметров viewport или другого статически вычислимого параметра
Во всех остальных случая параметр динамически вычислим.

Элементы характеризуются позицией и размером. Позиция - это выравнивание либо относительно родительского элемента, либо относительно родственных элементов

Позиция: left, top
Размеры: width, height

Элементы управления и чтения пусть имеют конечный набор градаций фиксированного размера в зависимости от параметров экрана устройства
пустоты пусть имеют процентный размер
